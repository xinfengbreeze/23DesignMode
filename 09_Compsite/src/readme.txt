结构型设计模式-----组合模式
这里的组合模式并不是Java中的"组合"

组合设计模式-----将对象组合成树形结构，以便表示"部分 与 整体"层次结构

组合模式
Component 接口 定义了 1、统一的功能 和 属性  2、获取和操作子节点的功能  3、（可选）获取和操作父节点的功能

单个节点  leaf   组合节点 composite

实际的例子   大树   文件夹与文件
一般和递归联合使用


组合方式分类：
透明方式：叶子类 和 组合类  的方法 包含的很全， 都定义在Component类中，但是可能实现自己不需要的方法
安全方式：叶子类 和 组合类  各自特有的方法定义在自己的类中，这样不需要实现自己不许要的方法

透明方式更好，这样在某些情况下叶子类和组合类可以合并到一起。而且对客户来说并不需要知道谁是叶子
